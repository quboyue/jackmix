Detailed Description

Stops an internal signal being sent outside if there is already a signal relayed from the outside to the inside.

The Setup:
A slider representing a value of an object but the object being far away. And there are several Sliders representing the objects value in different hosts. And all the sliders should get the changes from the others.
The Solution:
Connect every slider with the connections from an to the server and let the server update all the clients.
The Problem:
A slider being updated by the server sends the valueChanged-signal himself to the server, which in return notifies all the sliders, an infinite loop is born.
The REAL Solution:
Plug the UpdateFilter between the server-connection and the slider. Everytime the server sends updates the filter will block the signals from the inside.
The ports ( signals/slots ) to use:
Connect the externals to dataIn() / dataOut() and the internal, to-be-filtered objects to dataInInternal() / dataOutInternal().
